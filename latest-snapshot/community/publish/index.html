    Make a Sedona release¶

    This page is for Sedona PMC to publish Sedona releases.

    Warning

    All scripts on this page should be run in your local Sedona Git repo under master branch via a single script file.

    0. Prepare an empty script file¶

    1. In your local Sedona Git repo under master branch, run 
      echo '#!/bin/bash' > create-release.sh
chmod 777 create-release.sh
    2. Use your favourite GUI text editor to open create-release.sh.
    3. Then keep copying the scripts on this web page to replace all content in this script file.
    4. Do NOT directly copy/paste the scripts to your terminal because a bug in clipboard.js will create link breaks in such case.
    5. Each time when you copy content to this script file, run ./create-release.sh to execute it.

    1. Check ASF copyright in all file headers¶

    1. Run the following script: 
      #!/bin/bash
wget -q https://dlcdn.apache.org//creadur/apache-rat-0.15/apache-rat-0.15-bin.tar.gz
tar -xvf  apache-rat-0.15-bin.tar.gz
git clone --shared --branch master https://github.com/apache/sedona.git sedona-src
java -jar apache-rat-0.15/apache-rat-0.15.jar -d sedona-src > report.txt
    2. Read the generated report.txt file and make sure all source code files have ASF header.
    3. Delete the generated report and cloned files 
      #!/bin/bash
rm -rf apache-rat-0.15
rm -rf sedona-src
rm report.txt

    2. Update Sedona Python, R and Zeppelin versions¶

    Make sure the Sedona version in the following files are 1.5.0.

    1. https://github.com/apache/sedona/blob/master/python/sedona/version.py
    2. https://github.com/apache/sedona/blob/master/R/DESCRIPTION
    3. https://github.com/apache/sedona/blob/99239524f17389fc4ae9548ea88756f8ea538bb9/R/R/dependencies.R#L42
    4. https://github.com/apache/sedona/blob/master/zeppelin/package.json

    3. Update mkdocs.yml¶

    • Please change the following variables in mkdocs.yml to the version you want to publish.
      • sedona_create_release.current_version
      • sedona_create_release.current_rc
      • sedona_create_release.current_git_tag
      • sedona_create_release.current_snapshot
    • Then compile the website by mkdocs serve. This will generate the scripts listed on this page in your local browser.
    • You can also publish this website if needed. See the instruction at bottom.

    4. Stage and upload release candidates¶

    #!/bin/bash

git checkout master
git pull

rm -f release.*
rm -f pom.xml.*

echo "*****Step 1. Stage the Release Candidate to GitHub."

mvn -q -B clean release:prepare -Dtag=sedona-1.5.0-rc1 -DreleaseVersion=1.5.0 -DdevelopmentVersion=1.5.1-SNAPSHOT -Dresume=false -Penable-all-submodules -Darguments="-DskipTests"
mvn -q -B release:clean -Penable-all-submodules

echo "Now the releases are staged. A tag and two commits have been created on Sedona GitHub repo"

echo "*****Step 2: Upload the Release Candidate to https://repository.apache.org."

# For Spark 3.0 and Scala 2.12
mvn -q org.apache.maven.plugins:maven-release-plugin:2.3.2:perform -DconnectionUrl=scm:git:https://github.com/apache/sedona.git -Dtag=sedona-1.5.0-rc1 -Dresume=false -Darguments="-DskipTests -Dspark=3.0 -Dscala=2.12" -Dspark=3.0 -Dscala=2.12

# For Spark 3.0 and Scala 2.13
## Note that we use maven-release-plugin 2.3.2 instead of more recent version (e.g., 3.0.1) to get rid of a bug of maven-release-plugin,
## which prevent us from cloning git repo with user specified -Dtag=<tag>.
## Please refer to https://issues.apache.org/jira/browse/MRELEASE-933 and https://issues.apache.org/jira/browse/SCM-729 for details.
##
## Please also note that system properties `-Dspark` and `-Dscala` has to be specified both for release:perform and the actual build parameters
## in `-Darguments`, because the build profiles activated for release:perform task will also affect the actual build task. It is safer to specify
## these system properties for both tasks.
mvn -q org.apache.maven.plugins:maven-release-plugin:2.3.2:perform -DconnectionUrl=scm:git:https://github.com/apache/sedona.git -Dtag=sedona-1.5.0-rc1 -Dresume=false -Darguments="-DskipTests -Dspark=3.0 -Dscala=2.13" -Dspark=3.0 -Dscala=2.13

# For Spark 3.4 and Scala 2.12
mvn -q org.apache.maven.plugins:maven-release-plugin:2.3.2:perform -DconnectionUrl=scm:git:https://github.com/apache/sedona.git -Dtag=sedona-1.5.0-rc1 -Dresume=false -Darguments="-DskipTests -Dspark=3.4 -Dscala=2.12" -Dspark=3.4 -Dscala=2.12

# For Spark 3.4 and Scala 2.13
mvn -q org.apache.maven.plugins:maven-release-plugin:2.3.2:perform -DconnectionUrl=scm:git:https://github.com/apache/sedona.git -Dtag=sedona-1.5.0-rc1 -Dresume=false -Darguments="-DskipTests -Dspark=3.4 -Dscala=2.13" -Dspark=3.4 -Dscala=2.13

echo "*****Step 3: Upload Release Candidate on ASF SVN: https://dist.apache.org/repos/dist/dev/sedona"

echo "Creating 1.5.0-rc1 folder on SVN..."

svn mkdir -m "Adding folder" https://dist.apache.org/repos/dist/dev/sedona/1.5.0-rc1

echo "Creating release files locally..."

echo "Downloading source code..."

wget https://github.com/apache/sedona/archive/refs/tags/sedona-1.5.0-rc1.tar.gz
tar -xvf sedona-1.5.0-rc1.tar.gz
mkdir apache-sedona-1.5.0-src
cp -r sedona-sedona-1.5.0-rc1/* apache-sedona-1.5.0-src/
tar czf apache-sedona-1.5.0-src.tar.gz apache-sedona-1.5.0-src
rm sedona-1.5.0-rc1.tar.gz
rm -rf sedona-sedona-1.5.0-rc1

echo "Compiling the source code..."

mkdir apache-sedona-1.5.0-bin

cd apache-sedona-1.5.0-src && mvn -q clean install -DskipTests -Dspark=3.0 -Dscala=2.12 && cd ..
cp apache-sedona-1.5.0-src/common/target/sedona-*1.5.0.jar apache-sedona-1.5.0-bin/
cp apache-sedona-1.5.0-src/spark/target/sedona-*1.5.0.jar apache-sedona-1.5.0-bin/
cp apache-sedona-1.5.0-src/spark-shaded/target/sedona-*1.5.0.jar apache-sedona-1.5.0-bin/
cp apache-sedona-1.5.0-src/flink/target/sedona-*1.5.0.jar apache-sedona-1.5.0-bin/
cp apache-sedona-1.5.0-src/flink-shaded/target/sedona-*1.5.0.jar apache-sedona-1.5.0-bin/

cd apache-sedona-1.5.0-src && mvn -q clean install -DskipTests -Dspark=3.0 -Dscala=2.13 && cd ..
cp apache-sedona-1.5.0-src/spark/target/sedona-*1.5.0.jar apache-sedona-1.5.0-bin/
cp apache-sedona-1.5.0-src/spark-shaded/target/sedona-*1.5.0.jar apache-sedona-1.5.0-bin/

cd apache-sedona-1.5.0-src && mvn -q clean install -DskipTests -Dspark=3.4 -Dscala=2.12 && cd ..
cp apache-sedona-1.5.0-src/spark/target/sedona-*1.5.0.jar apache-sedona-1.5.0-bin/
cp apache-sedona-1.5.0-src/spark-shaded/target/sedona-*1.5.0.jar apache-sedona-1.5.0-bin/

cd apache-sedona-1.5.0-src && mvn -q clean install -DskipTests -Dspark=3.4 -Dscala=2.13 && cd ..
cp apache-sedona-1.5.0-src/spark/target/sedona-*1.5.0.jar apache-sedona-1.5.0-bin/
cp apache-sedona-1.5.0-src/spark-shaded/target/sedona-*1.5.0.jar apache-sedona-1.5.0-bin/

tar czf apache-sedona-1.5.0-bin.tar.gz apache-sedona-1.5.0-bin
shasum -a 512 apache-sedona-1.5.0-src.tar.gz > apache-sedona-1.5.0-src.tar.gz.sha512
shasum -a 512 apache-sedona-1.5.0-bin.tar.gz > apache-sedona-1.5.0-bin.tar.gz.sha512
gpg -ab apache-sedona-1.5.0-src.tar.gz
gpg -ab apache-sedona-1.5.0-bin.tar.gz

echo "Uploading local release files..."

svn import -m "Adding file" apache-sedona-1.5.0-src.tar.gz https://dist.apache.org/repos/dist/dev/sedona/1.5.0-rc1/apache-sedona-1.5.0-src.tar.gz
svn import -m "Adding file" apache-sedona-1.5.0-src.tar.gz.asc https://dist.apache.org/repos/dist/dev/sedona/1.5.0-rc1/apache-sedona-1.5.0-src.tar.gz.asc
svn import -m "Adding file" apache-sedona-1.5.0-src.tar.gz.sha512 https://dist.apache.org/repos/dist/dev/sedona/1.5.0-rc1/apache-sedona-1.5.0-src.tar.gz.sha512
svn import -m "Adding file" apache-sedona-1.5.0-bin.tar.gz https://dist.apache.org/repos/dist/dev/sedona/1.5.0-rc1/apache-sedona-1.5.0-bin.tar.gz
svn import -m "Adding file" apache-sedona-1.5.0-bin.tar.gz.asc https://dist.apache.org/repos/dist/dev/sedona/1.5.0-rc1/apache-sedona-1.5.0-bin.tar.gz.asc
svn import -m "Adding file" apache-sedona-1.5.0-bin.tar.gz.sha512 https://dist.apache.org/repos/dist/dev/sedona/1.5.0-rc1/apache-sedona-1.5.0-bin.tar.gz.sha512

echo "Removing local release files..."

rm apache-sedona-1.5.0-src.tar.gz
rm apache-sedona-1.5.0-src.tar.gz.asc
rm apache-sedona-1.5.0-src.tar.gz.sha512
rm apache-sedona-1.5.0-bin.tar.gz
rm apache-sedona-1.5.0-bin.tar.gz.asc
rm apache-sedona-1.5.0-bin.tar.gz.sha512
rm -rf apache-sedona-1.5.0-src
rm -rf apache-sedona-1.5.0-bin

    7. Failed vote¶

    If a vote failed, do the following:

    1. In the vote email, say that we will create another release candidate.
    2. Restart from Step 3 Update mkdocs.yml. Please increment the release candidate ID (e.g., 1.5.0-rc2) and update sedona_create_release.current_rc and sedona_create_release.current_git_tag in mkdocs.yml to generate the script listed on this webpage.

    9. Release Sedona Python and Zeppelin¶

    You must have the maintainer privilege of https://pypi.org/project/apache-sedona/ and https://www.npmjs.com/package/apache-sedona

    To publish Sedona pythons, you have to use GitHub actions since we release wheels for different platforms. Please use this repo: https://github.com/jiayuasu/sedona-publish-python

    #!/bin/bash

wget https://github.com/apache/sedona/archive/refs/tags/sedona-1.5.0-rc1.tar.gz

cd zeppelin && npm publish && cd ..
rm -rf apache-sedona-1.5.0-src

    #!/bin/bash

    Then submit to CRAN using this web form.

    11. Publish the doc website¶

    Prepare the environment and doc folder¶

    1. Check out the 1.5.0 Git tag on your local repo.
    2. Read Compile documentation website to set up your environment. But don't deploy anything yet.
    3. Add the download link to Download page.
    4. Add the news to docs/index.md.

    Generate Javadoc and Scaladoc¶

    Run the following script to build Javadoc and Scaladoc of sedona modules and move them to docs/api/javadoc directory.

    #!/bin/bash

mvn -q clean install -DskipTests
rm -rf docs/api/javadoc && mkdir docs/api/javadoc
mv core/target/apidocs docs/api/javadoc/core
mv viz/target/apidocs docs/api/javadoc/viz
mv sql/common/target/site/scaladocs docs/api/javadoc/sql

    Please do not commit these generated docs to Sedona GitHub.

    Compile R html docs¶

    From GitHub Action docs workflow, find generated-docs in the tagged commit. Download it and copy this folder docs/api/rdocs to the same location of the Sedona to-be-released source repo.

    Deploy the website¶

    1. Run mike deploy --push --update-aliases 1.5.0 latest. This will deploy this website to Sedona main repo's gh-page. But Sedona does not use gh-page for hosting website.
    2. Check out the master branch.
    3. Git commit and push your changes in download.md and index.md to master branch. Delete all generated docs.
    4. Check out the gh-page branch.
    5. In a separate folder, check out GitHub sedona-website asf-site branch
    6. Copy all content to in Sedona main repo gh-page branch to Sedona website repo asf-site branch.
    7. Commit and push the changes to the remote asf-site branch.
